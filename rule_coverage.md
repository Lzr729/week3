# Rule Coverage

## 1. 方法定位

最终批量流程以规则抽取为主，不在运行时调用LLM。规则承担章节定位、事件包切分、日期和数值提取、单位归一化、表格解析、主体过滤以及复核原因生成。

覆盖率与准确率分开报告：

- 覆盖率回答候选事件包是否产生结构化结果或错误记录；
- 字段准确率回答自动字段是否与人工Gold一致；
- `REVIEW_REQUIRED`表示已覆盖但需要复核，不表示遗漏。

## 2. 已覆盖的章节和标题

候选定位规则主要处理以下标题或正文锚点：

- 历史沿革；
- 股本形成及变化；
- 历次增资；
- 股票发行或定向发行；
- 股权转让；
- 整体变更；
- 吸收合并；
- 本次变更后股权结构；
- 本次发行前后股本结构；
- 前十名股东；
- 股权关系图。

## 3. subscription_flow覆盖

较稳定：

- “注册资本由X增至Y”；
- “新增注册资本X万元”；
- 单一认购方货币增资；
- 多参与方逐行列示认缴额和出资额；
- 股票定向发行的发行股数、价格、募集金额和资本公积；
- 吸收合并形成的新增注册资本；
- 决议、协议、验资、工商和挂牌日期的常见表达。

已验证代表场景：

- 三联锻造：单一认购方现金增资；
- 赛分科技：多参与方现金增资；
- 三协电机：定向发行；
- 黄山谷捷：吸收合并增资；
- 星图测控：现有股东认购增资。

主要失败：

- 多批次工商变更应合并为一个事件；
- 同节混合增资和股权转让；
- 多组不同价格；
- 参与方金额需要跨段落汇总；
- 表格或正文顺序被PDF文本层打乱。

## 4. share_transfer_flow覆盖

较稳定：

- “A将其持有的发行人股权转让给B”；
- 一对一或一对多转让；
- 明确披露转让比例、注册资本、股份数量或对价；
- 零对价内部重组；
- 协议日期、股东决定日期和工商变更日期；
- 转让后股权结构表。

主体过滤：

- 被转让标的必须是发行人或发行人前身；
- 上层公司、员工持股平台内部份额和其他企业股权不直接作为发行人股权转让；
- `target_is_issuer`和`target_confirmation`保留主体确认依据。

已知边界：

- 简称与完整公司名称的规范化；
- 一段文字中连续列出多笔转让；
- 转让和增资使用同一协议；
- 转让前后跨页结构表。

## 5. equity_snapshot覆盖

基础版支持：

- `序号—股东名称—持股数量—持股比例`；
- `序号—股东名称—出资额—出资比例`；
- 万股、股、万元和百分比归一化；
- 合计行识别；
- 发行前或变更后结构；
- 发行前后股本结构中的单侧快照。

已知边界：

- 跨页表格缺少重复表头；
- 多级表头；
- 发行前和发行后同时出现两套数量及比例；
- 股权关系图而非文本表格；
- OCR或文本层列顺序错乱；
- 单页存在多个不同口径的股东表。

## 6. 八家公司批量覆盖

最终批量结果：

- 8家公司全部运行；
- 3个模块共24个审计单元；
- 候选事件包49个；
- 结构化记录49条；
- 抽取器错误0；
- 未覆盖候选事件包0；
- 48条记录需要人工复核；
- 9条记录未通过JSON Schema；
- 复核队列57行。

因此，流水线达到“无静默遗漏”的覆盖目标，但不宣称49条记录均达到Gold准确度。

## 7. 失败处理

- 结构解析失败：写入抽取日志；
- Schema失败：写入校验日志并进入高优先级复核；
- 数值不一致：保留expected、actual和difference；
- 字段缺失：Cross-check标记为`SKIP`并保留原因；
- 低置信度：记录`needs_manual_review=true`和`review_reasons`；
- 候选事件包没有结构化结果或错误记录：标记`COVERAGE_GAP`。
